Build a new “Internal Draft System” for the Facebook Ads duplication platform.

Core idea:
Do NOT immediately publish duplicated or converted Meta Ads objects.

Instead:

- Save duplicated/converted Campaigns, Ad Sets, and Ads as INTERNAL DRAFTS first
- Allow users to fully edit draft data before publishing to Meta
- Only create real Meta objects AFTER final confirmation

The goal is to avoid Meta API field locking issues after object creation.

IMPORTANT:
This system must be implemented as a NEW isolated feature and MUST NOT break the existing duplication or conversion workflows.

==================================================
PROBLEM TO SOLVE

After publishing Meta objects:

- many fields become immutable
- some optimization fields cannot be changed
- some promoted_object settings become locked
- some placements/objectives become uneditable

Examples:

- optimization_goal
- billing_event
- promoted_object
- attribution_setting
- conversion_location
- objective
- CTA type
- identity settings

This causes major workflow limitations after duplication or conversion.

==================================================
SOLUTION

Instead of publishing immediately:

NEW FLOW:

Duplicate / Convert
↓
Save as INTERNAL DRAFT
↓
Allow full editing
↓
Validate compatibility
↓
Preview final payload
↓
Publish ONCE to Meta API

==================================================
DRAFT SYSTEM REQUIREMENTS

Create internal draft models for:

- DraftCampaign
- DraftAdSet
- DraftAd

Drafts should:

- exist only in the platform database initially
- NOT create Meta objects yet
- support full editing before publish
- support validation before publish

==================================================
DRAFT EDITING CAPABILITIES

Users should be able to freely edit:

- objective
- optimization_goal
- billing_event
- promoted_object
- targeting
- placements
- creatives
- CTA
- tracking
- naming
- budget
- schedules

before publishing.

==================================================
TEMPORARY ID SYSTEM

Because Meta objects do not exist yet:

- create temporary internal IDs

Examples:

- temp_campaign_1
- temp_adset_1
- temp_ad_1

When publishing:

- map temporary IDs to real Meta IDs

==================================================
VALIDATION SYSTEM

Before publish:
Run validation checks for:

- objective compatibility
- optimization_goal compatibility
- promoted_object validity
- targeting validity
- placement compatibility
- deprecated fields
- missing required fields

If invalid:

- block publish
- show validation errors
- highlight problematic fields

==================================================
PREVIEW SYSTEM

Before publishing:
Show:

- final Campaign payload
- final Ad Set payloads
- final Ad payloads
- changed fields
- removed fields
- auto-generated defaults

Add:

- Diff viewer
- Human-readable summary
- Validation warnings

==================================================
PUBLISH SYSTEM

When user clicks Publish:

1. Create Campaign
2. Map new campaign_id
3. Create Ad Sets
4. Map new adset_ids
5. Create Ads
6. Save Meta IDs back into draft records

If failure occurs:

- stop safely
- show detailed errors
- allow retry

==================================================
IMPORTANT SAFETY REQUIREMENT

ALL newly published Meta objects MUST ALWAYS remain:

{
"status": "PAUSED"
}

Apply to:

- Campaigns
- Ad Sets
- Ads

The system must NEVER auto-publish active ads.
